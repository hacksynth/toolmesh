# 安全策略

## 支持范围

当前仓库尚未建立长期维护分支。现阶段仅默认维护 `main` 分支上的最新代码，历史快照和未发布分支不承诺单独回溯修复。

## 报告漏洞

如果你发现了可能影响用户安全的问题，请不要直接创建公开 Issue。

建议按以下顺序处理：

1. 如果仓库启用了 GitHub 私密安全报告功能，优先使用该渠道
2. 如果尚未启用，请通过 GitHub 私下联系仓库维护者，并在公开渠道披露前等待初步响应

报告中尽量包含以下信息：

- 漏洞类型与影响范围
- 受影响的命令、runtime 或版本
- 最小复现步骤
- 可能的利用条件、日志或截图
- 如有的话，临时缓解方案或修复建议

## 响应方式

- 维护者会尽量先确认问题是否可复现
- 修复通常会优先落到 `main`
- 在补丁发布或风险被明确缓解之前，请避免公开披露细节

## 非安全问题

普通功能缺陷、兼容性问题或体验改进，请继续使用公开 Issue 模板提交。
