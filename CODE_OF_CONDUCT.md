# 行为准则

## 我们的目标

`toolmesh` 希望成为一个友善、专业、可协作的开源项目。无论贡献者的经验水平、背景、身份或使用场景如何，社区交流都应以尊重、事实和建设性反馈为基础。

## 我们鼓励的行为

- 尊重不同经验层次，耐心沟通
- 在讨论中聚焦事实、复现步骤和可执行建议
- 对代码和设计提出具体、可验证的反馈
- 在出现分歧时优先澄清目标、约束和权衡

## 不可接受的行为

- 人身攻击、侮辱、骚扰、歧视或威胁
- 恶意挑衅、刷屏、故意破坏讨论秩序
- 在未经同意的情况下公开他人的隐私、凭据或安全信息
- 明知会伤害协作环境却持续重复的不当行为

## 适用范围

本准则适用于仓库内外所有与 `toolmesh` 相关的公共协作场景，包括但不限于 Issue、Pull Request、代码评审、讨论串和发布说明。

## 维护者职责

维护者有责任解释并执行本准则，也有权对不当行为采取适当措施，例如编辑或删除评论、锁定讨论、拒绝贡献或限制参与权限。

## 报告方式

如果你遇到违反本准则的行为，请通过 GitHub 私下联系仓库维护者。涉及漏洞、凭据泄露或其他安全问题时，请改走 [`SECURITY.md`](SECURITY.md) 中的流程。

## 执行原则

维护者会尽量结合上下文处理问题，并优先选择与风险相匹配的响应方式，例如提醒、警告、临时限制或永久移除参与资格。
